# project-structure-standard — Python 项目结构统一规范

## 一句话

散乱的项目目录按统一模板重组：根目录只放调度入口，功能代码归子目录，配置集中，调试隔离。

---

## 适用场景

- 项目根目录几十上百个文件混在一起，分不清生产代码和调试产物
- 你要快速找到：入口在哪、配置在哪、核心逻辑在哪
- 多个项目需要统一的目录规范

## 解决的问题

| 问题 | 方案 |
|------|------|
| 文件散落根目录 | 按职责划分子目录 |
| 配置散落各处 | 全部集中到 `config.py` |
| 调试探针混入生产 | 所有探针/截图归入 `dev/` |
| 不知道哪里能改 | `★` 标记标明唯一修改点 |

---

## 通用目录模板

```
项目名/
│
├── core/                  ★ 不动：核心框架/基类
├── adapters/              ★ 不动：平台适配器（如果有）
├── post/                  ★ 不动：后处理/回调脚本
├── dashboard/             ★ 不动：看板/可视化（如果有）
│
├── tools/                 ★ 独立功能工具
│   ├── tool_a.py
│   └── tool_b.py
│
├── bin/                   ← 批处理/快捷脚本
├── config/                ← 配置文件（密钥、cookies等敏感信息）
│
├── dev/                   ← 调试探针/截图收容处（不参与生产）
│   ├── probes/            ← 探针脚本 _probe*.py
│   ├── debug/             ← 调试脚本 debug*.py
│   ├── records/           ← 运行记录、中间输出
│   ├── explores/          ← 实验性脚本 explore*.py test*.py
│   └── screenshots/       ← 截图 *.png
│
├── main.py                ★ 调度入口（看懂调用顺序就行）
├── config.py              ★ ★ ★ 你改配置的地方 ★ ★ ★
│
├── data/                  ← 数据文件（gitignore）
├── logs/                  ← 运行日志（gitignore）
│
└── README.md              ← 项目说明
```

---

## 核心规则

1. **根目录只放调度入口** — `main.py`（或 `daily_export.py`）是唯一根级脚本
2. **配置只改 `config.py`** — 路径、参数、开关全部集中
3. **调试文件不进根目录** — 所有 `_probe*` `debug*` `explore*` `test*` 截图一律放入 `dev/`
4. **★ 标记告诉你哪里能改** — 带 `★` 的文件是你会碰到的，其他别动
5. **后处理归入 `post/`** — 流水线里的后处理步骤统一放 `post/`
6. **独立功能归入 `tools/`** — 不在主流程里的工具脚本放 `tools/`

## ★ 标记说明

| 标记 | 含义 |
|------|------|
| `★ ★ ★` | 你唯一需要改的地方 |
| `★` | 看懂调用顺序就行，不用改 |
| `★ 不动` | 基类/框架代码，不要碰 |

---

## 迁移步骤（复制构建法）

不改当前运行版本，复制一份重组，验证后再切换：

```
1. cp -r 原项目/ 项目2/        ← 复制副本
2. 在项目2/ 里重组结构           ← 移动文件到子目录
3. 更新 main.py 里的路径引用     ← 如有 import/子进程路径变化
4. py run.py -p xxx 试跑       ← 验证副本
5. mv 原项目/ archive/          ← 原版归档
6. mv 项目2/ 原项目名            ← 副本上位（路径不变）
```

**关键原则：** 不删原版，副本验证，最后改个名字就完成切换。计划任务和配置引用都不受影响。

---

## 调试文件隔离流程

当你要写探针/截图调试时：

1. 脚本命名：`dev/probes/你描述的问题.py`
2. 截图命名：`dev/screenshots/你描述的问题.png`
3. 调试完后不删除，留在原处供下次参考
4. 根目录不出现任何 `_probe*` `debug*` 文件
